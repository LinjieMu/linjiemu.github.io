1. 算法流程

一般情况下，一棵完整的决策树由一个根结点、若干个内部结点和叶结点组成。其中，根结点包含样本的全集，在属性的划分下，生成不同的子结点，所有的叶结点就是最终划分的结果。决策树学习的目的就是为了生成一棵泛化能力强的树。

其基本算法流程如下：

输入：训练集 D={(x1,y1),(x2,y2),…,(xm,ym)}D=\{(x_1,y_1),(x_2,y_2),\dots,(x_m,y_m)\}D={(x1​,y1​),(x2​,y2​),…,(xm​,ym​)};

过程：函数TreeGenerate(D,A)

1： 生成结点node；

2： if D中的样本全部属于同一类别C then

3： 将node标记为C类叶结点；return

4： end if

5： if A=∅A=\emptysetA=∅ OR D 中样本在 A 上取值相同 then

6： 将node标记为叶结点，其类别标记为D中样本数目最多的类；return

7： end if

8： 从A中选择最优划分属性a∗a_*a∗​；

9： for a∗a_*a∗​ 的每一个值a∗va_*^va∗v​ do

10： 为node生成一个分支；令DvD_vDv​表示DDD中在a∗a_*a∗​上取值为a∗va_*^va∗v​的样本子集；

13： else

15： end if

输出：以node为根节点的决策树

注意三种返回结果：

  1. 当前结点包含的样本全属于相同的类别：设定该相同类别为当前结点的值；
  2. 当前的属性集合为空，或者样本在所有属性上取值相同：设定样本所含最多类为当前结点的值；
  3. 当前结点包含的样本集合为空：设点父结点样本所含最多类为当前结点的值；

核心步骤：寻找当前集合A中选择最优划分属性a∗a_*a∗​。

2. 划分选择

随着决策树划分的不断进行，我们希望分支结点所含样本尽可能属于相同的类别，即结点的纯度越来越高。在此，我们以贪心的思想，选择可以使当前结点纯度最大化的属性，来进行下一次结点划分。下面介绍几种描述结点纯度（增量）的指标。

1. 信息增益

假定当前集合DDD中第kkk类样本所占比例为pk(k=1,2,…,∣y∣)p_k(k=1,2,\dots,|\mathscr{y}|)pk​(k=1,2,…,∣y∣)，则DDD的信息熵定义为：

Ent(D)=−∑k=1∣y∣pk log2pkEnt(D)=-\sum_{k=1}^{|\mathscr{y}|}p_k\ log_2p_k Ent(D)=−k=1∑∣y∣​pk​ log2​pk​

Ent(D)Ent(D)Ent(D)值越低，当前集合DDD的纯度越高。

假设离散属性a有V个可能的取值{a1,a2,…,aV}\{a^1,a^2,\dots,a^V\}{a1,a2,…,aV}，若使用a对样本集合D进行划分，则可以产生V个分支结点。记集合D在属性a上取值为ava^vav所产生的样本集合为DvD^vDv，信息增益的定义为：

Gain(D,a)=Ent(D)−∑v=1V∣Dv∣∣D∣Ent(Dv)Gain(D,a) =Ent(D)-\sum^V_{v=1}\frac{|D^v|}{|D|}Ent(D^v) Gain(D,a)=Ent(D)−v=1∑V​∣D∣∣Dv∣​Ent(Dv)

一般而言，信息增益越大，意味着属性a来进行划分对决策树带来的纯度增益越大。因此，我们有理由选择当前结点拥有最大的信息增益的属性进行划分，即

a∗=arg⁡max⁡a∈AGain(D,a)a_* = \arg \max_{a\in A}Gain(D,a) a∗​=arga∈Amax​Gain(D,a)

2. 增益率

单纯的使用信息增益可能会造成一些问题。不难发现，信息增益准侧对可取值数目较多的属性有所偏好。而这种偏好可能造成不良影响，降低决策树的泛化能力，无法对新的样本进行有效的预测。因此，我们提出增益率的概念。

Gain_ratio(D,a)=Gain(D,a)IV(a)Gain\_ratio(D,a)=\frac{Gain(D,a)}{IV(a)} Gain_ratio(D,a)=IV(a)Gain(D,a)​

式中，

IV(a)=−∑v=1V∣Dv∣∣D∣log2∣Dv∣∣D∣IV(a)=-\sum_{v=1}^{V}\frac{|D^v|}{|D|}log_2\frac{|D^v|}{|D|} IV(a)=−v=1∑V​∣D∣∣Dv∣​log2​∣D∣∣Dv∣​

称为属性a的固有值。属性a的可取值数目越多，固有值越大。

注意，增益率准则对可取值数目较少的属性有所偏好。因此在选择属性时，我们综合这两种概念：先从候选划分属性中选择信息增益率高于平均水平的属性，再从中选择增益率最高的。

3. 剪枝处理

1. 预剪枝

  1. 定义：在决策树生成过程中，对每个结点在划分，若当前结点的划分不能带来泛化性能的提升，则停止划分并将当前结点标记为叶结点。
  2. 方法：可以使用留出法，将已知数据集划分为两个部分，训练集和测试集。用测试集计算当前结点展开前和展开后的准确率变化。若当前结点的展开没有带来泛化性能的提升，则不进行展开。
  3. 评价：
    1. 降低了过拟合的风险；
    2. 减小了决策树的训练时间和测试时间开销；
    3. 也给决策树带来了欠拟合的风险。

2. 后剪枝：

  1. 定义：先从训练集生成一棵完整的决策树，然后字底向上地对非叶结点进行考察，若将该结点对应的子树替换为叶结点能带来决策树泛化性能提升，则将该子结点替换为叶结点。
  2. 评价：
    1. 后剪枝决策树欠拟合风险小，泛化能力往往高于预剪枝决策树；
    2. 训练时间开销比预剪枝树要大得多。
