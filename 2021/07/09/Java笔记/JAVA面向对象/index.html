Java学习笔记(二):面向对象

发表于2021-07-09|更新于2021-07-09|Java
|字数总计:2.4k|阅读时长:7分钟|阅读量:

1.基本概念

1.对象（Object）

  • 状态：对象本身的信息（变量）

  • 行为：实现对象的访问/对象的操作（方法）

  • 对象是变量和相关方法的集合，对象是具有责任的实体

  • 程序是对象的集合，对象之间通过发送消息完成任务

2.类（Class)

  • 类是一种原型，一种抽象，一种共性，一个模板
  • 对象与类的关系：
    • 对象是具体的，是其所属的类的某个具体实例
    • 必须先有类才能有对象
    • 类是静态的，对象是动态的

3.封装性

  • 对象的某些状态被保护/隐藏，其他对象无法直接访问
  • 其他对象只能通过该类的方法（接口，interface）访问被保护的状态
  • 好处：
    • 模块化：每个对象的源文件相互独立的，可以被不同的程序调用
    • 信息隐藏：被定义为公共属性的数据和方法才能被其他对象访问，否则不能访问

4.继承性

  • 目的：实现代码复用
  • Java中所有类的基类：Object类
  • 子类继承基类所有的数据成员和方法，同时子类可以有新的数据成员和方法
  • Java中仅支持单一继承，同时采取接口变相实现多重继承

5.多态性

  1. 方法的重写（overwrite）
    • 多个子类从同一个基类继承（extends）而来
    • 每个子类都重写了父类的某个方法
    • 被重写的方法在不同的子类中具有不同形式
  2. 方法的重载（override）
    • 同名方法拥有不同的形参列表

2.类的描述

1.类的修饰符

  • public：公共类，可以被其他任何类使用
  • 无默认修饰符：仅能被同一个包内的其他类引用
  • abstract：抽象类，不能被实例化/创建对象
  • final：最终类，不能被继承

2.继承和接口

  • extends：一个类继承另一个类（仅一个）
  • implements：一个类实现一些（可以是多个）接口的方法，用以变相实现多继承

3.包的概念（package）

  • 源文件的位置在"D:\src\org\mycompany\Test.java"（注意：包的文件层次关系和类名与文件名的关系）

  • 编译方法1：在源文件所在文件夹下编译

  • 编译方法2：在根目录下编译

  • 生成的字节码文件与源文件在同一文件夹下

  • 运行class的方法：在根目录下运行，注意运行时不加文件后缀

  • 作用：

    • 防止同名类发生冲突

3.类成员和成员方法的访问修饰符

  • 公共权限public：能被所有类访问

  • 默认权限不写：可以被同一个包中的所有类访问

  • 私有权限private：只能被该类自身访问

  • 保护权限protected：只能被同一包中的类、该类的子类访问

4.类成员和成员方法的非访问修饰符

  • 静态成员/方法 static
    • 独立于类的对象，可以直接通过类进行访问
    • 静态方法只能访问静态变量或方法，反之不亦然
    • 静态变量/方法每个类只有一份
  • 最终成员/方法final
    • 对于成员变量：程序执行过程中不允许被改变
    • 对于成员方法：不允许被子类重写
  • 抽象方法abstract
    • 只有方法说明，没有方法体
  • 本地方法native
    • 其他语言实现的方法功能
  • 同步方法synchronized
    • 多线程程序中的协调和同步

5.方法的传参

  • 区分实参和形参
  • 实参列表必须与形参列表一致，否则先考虑类型转化，如果不能转化则出错
  • Java仅支持值传递

6. 变量的作用域

  • 成员变量：从类声明开始到类结束为止
  • 局部变量：
    • 方法中：声明处开始，方法结束为止
    • 复合语句中：声明处开始，复合语句结束为止
    • 形参的作用域：方法内
  • 局部变量与成员变量重名时，后者被屏蔽

7.方法的重载（overload）

  • 一个类中方法名相同，但形参列表不同

8.方法的重写（override）

  • 子类重写父类的方法

9.构造方法

  1. 对象创建通过构造方法来实现，其可以在对象被创建时，做一些初始化工作
  2. 构造方法的名字与类名相同
  3. 构造方法无返回值
  4. 构造方法可以 有多个，但形参列表不同（方法重载）

10.默认构造方法

  • 如果没有编写任何构造方法则系统会自动创建一个无参数的构造方法
  • 如果类存在有参数的构造方法，则系统不会自动创建默认构造方法

4.抽象类

  • 抽象类中含有抽象方法
  • 抽象方法：仅仅写出来了方法的修饰符、名字、参数、返回类型等，没有函数体
  • 只有抽象类中抽象方法
  • 抽象类不能被实体化，即创建对象
  • 抽象类可以具有普通方法
  • 子类继承抽象类时必须重写抽象方法，否则仍为抽象类
  • 多态与抽象类：
    • 父类无法预期其各个子类的某一行为的具体表现形式
    • 抽象类为子类规定了同一的形式

  • 将对象从内存中清除（释放内存空间）
  • Java会周期性地释放不再被引用的对象，自动完成
  • 手动完成System.gc()

3.对象的使用

  • 关键词this

    • 指向当前对象
    • 应用1：增强程序的可读性
    • 应用2：对同一个对象的执行多次方法调用
    • 应用3：在一个构造方法中调用另一个构造方法

  • 判断对象是否为空

  • 判断对象的真实类型

4.类的继承和多态

1.类的继承

  • 基本内容
    • 子类可以调用父类的方法和变量，子类可以增加父类中没有的方法和变量
    • 子类继承父类中的构造方法
    • 类成员访问修饰符与继承的关系
  • 调用子类的构造方法时，下调用其父类的构造方法
    • 显示调用
    • 隐式调用
      • 没有写调用父类构造方法的代码
      • 实际是调用父类的无参数构造方法
  • super可以用来引用父类中的（被覆盖的）方法、（被隐藏的）变量及构造方法
    • 注意：子类显示调用父类构造方法时，super必须是子类构造方法的第一条
  • 方法重写：
    • 重写：子类和父类的方法名、返回类型和参数均相同（方法可见性可以不同）
    • final/private方法不能重写
    • 子类中重写的方法不能降低父类中被重写方法的可见性
    • 子类中重写的方法产生的异常不能超过父类中被重写的方法
  • 子类对象与父类对象
    • 子类对象可以被视为其父类的一个对象
    • 父类对象不能当作某一个子类对象

5.接口和包

  • 接口是对abstract类的进一步扩展

  • 接口中的方法都是未实现的（类似于抽象方法），目的是在实现接口的类之间建立一种协议

  • 接口中的变量都是常量

  • 定义方法：

  • 一个类符合某个或一组接口，利用implements

  • 接口名修饰

    • public：无任何访问限制
    • 无修饰：仅限于本包中

  • 接口变量默认都是public static final

  • 接口方法：

    • 无修饰
    • 但在实现接口方法的类中，修饰符为public

  • 接口的继承extends

    • 用以将多个接口合并为一个新接口
